# Data Domain

> Data engineering, analytics, and storage patterns.

## Quick Summary

This domain covers working with data at scale: pipelines, storage, analytics, and data quality.

## Files in This Domain

No playbooks currently in this domain. Contribute following `meta/contributing.md`.

## Key Principles

1. **Data quality at the source** - Garbage in, garbage out
2. **Schema evolution** - Plan for change
3. **Idempotent pipelines** - Safe to re-run
4. **Document lineage** - Know where data comes from
5. **Privacy by design** - Handle PII carefully

## When to Consult This Domain

- Building data pipelines
- Choosing data storage solutions
- Implementing analytics tracking
- Ensuring data quality
- Planning data architecture

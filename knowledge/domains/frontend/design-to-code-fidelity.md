# Design-to-Code with Machine-Verified Fidelity

> Convert an HTML design mock into a production component app where fidelity is proven by numbers, not eyes — and the LLM is paid only for semantics, never for geometry.

## TL;DR

- **Rendered DOM is the ground truth.** Never parse a design tool's source format; render it headless and extract the computed-layout tree. Geometry travels as numbers end to end.
- **Transform-then-refactor-under-lock beats generate-then-converge.** Syntactic HTML→JSX transform starts at fidelity; the LLM refactors without breaking it. Pilot result: 1 finding, 0 fix cycles, vs a 52-defect burn-down loop on the prose-driven route.
- **Tree-diff is the primary gate** (names element + property + delta — pre-diagnosed defects); **pixel-diff is only the backstop** (catches glyphs, z-order, clipping that computed styles can't see).
- **Never scale geometry proportionally.** Tokens, type, and radii are not scale-invariant. Frame mismatches are eliminated at the reference (re-capture the mock reshaped/retinted) or absorbed by layout intent, never by ratios.
- **LLM does componentization and behavior only, under a green gate.** Everything mechanical moves by script at zero marginal cost.
- **At app scale the pipeline is a compiler, not an agent-per-screen.** Batch the deterministic ~80% (extract → transform → one palette→token map → responsive scaffold) and verify every screen against ONE long-lived server; the LLM is only the judgment tail — ambiguous colors, readability, screenshot-caught bugs.
- **A fixed desktop mock underspecifies responsive.** Don't pixel-clone its fixed styles (that ships anti-responsive CSS). Build fluid; the mock is the desktop reference; verify desktop-fidelity within tolerance **plus** a mobile no-overflow gate the one-viewport tree-diff can't provide.

## Decision Guide

| Scenario                                                                                                                           | Approach                                                                                                                                                                                                          | Why                                                                                                                                                                                          |
| ---------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Design exists as HTML/CSS (hand-mock or AI design tool)                                                                            | Headless-render it; extract computed tree + DOM dump with stamped element ids; transform mechanically to JSX                                                                                                      | Lossless channel for every mechanical fact; LLM re-deriving padding from prose is the cost leak                                                                                              |
| Design source is opaque/self-extracting (e.g. Claude Design `.dc.html`, base64-packed exports)                                     | Never parse the file; render and extract. The rendered DOM is the only stable contract                                                                                                                            | Internal formats are undocumented and churn; computed styles are a W3C-stable API                                                                                                            |
| Export is a variant gallery / inline-styles-only handoff (multiple competing designs per screen, no state list, own design system) | The fidelity engine still applies, but PREP first: pick one winner per screen, author a states list, build a value-map + kit-component reconciliation. Budget heavy inline→CSS hoisting                           | The engine is input-agnostic; reliability is bounded by input readiness. Unresolved design + inline styles + no encoded flows shrink the transform's cheap win and leave behavior unverified |
| Mock chrome ≠ app shell (sidebar widths, colors, frames)                                                                           | Eliminate at the reference: inject shell dimensions/brand tokens into the mock at capture time, re-capture                                                                                                        | Cheapest fix; keeps pixel-diff defined; one CSS injection vs N app-side exceptions                                                                                                           |
| Container widths can't be matched                                                                                                  | Split properties: invariants (tokens, gaps, padding, type, color, radius, column count) diffed exactly; fluids (text-bearing widths/x) diffed by rule                                                             | Pixel-perfect is undefined across widths; rule-fidelity is the honest contract                                                                                                               |
| Tempted to scale positions by container ratio                                                                                      | Don't. Exception: fixed-aspect visual compositions (charts, hero art) via `aspect-ratio`/% CSS                                                                                                                    | 24px × 0.82 is off every token ladder; scaled type is illegible; it's the artboard transform, not the UI transform                                                                           |
| Verifying the build matches                                                                                                        | Tree-diff paired by stamped ids, ±1px boxes, exact colors post-token-map; pixel score as final backstop                                                                                                           | Tree-diff says _why_ (element, property, delta); pixels only say _that_                                                                                                                      |
| Componentizing flat transformed JSX                                                                                                | LLM pass with the gate green before and after: "refactor, don't change rendered output"                                                                                                                           | Inverts economics: LLM must _not break_ fidelity (cheap) instead of _achieve_ it (expensive loop)                                                                                            |
| Multi-state screens (flows, runners, toggles)                                                                                      | Encode states as replayable step scripts (click selectors); same script drives mock and app; tree-diff per state                                                                                                  | Transform preserves class names, so one step script is both the state spec and the behavior-parity test                                                                                      |
| Deciding whether a kit component can replace mock markup                                                                           | Computable test: kit component's computed subtree matches the mock subtree within tolerance                                                                                                                       | Removes a judgment call from the paid step                                                                                                                                                   |
| App must be responsive, but the mock is a fixed desktop canvas                                                                     | Build fluid-native (grid collapse at a breakpoint, the kit shell's own responsive/pinned mode) — don't clone fixed styles. Mock = desktop spec; gate = desktop tree-diff (tolerant) + mobile no-overflow          | Cloning fixed inline styles ships anti-responsive CSS (real case: 188px mobile overflow). The mock defines one viewport; mobile is graceful degradation, not a designed screen               |
| Rebuilding a whole app (many screens), not one                                                                                     | Batch the deterministic pipeline once (extract-all → transform-all → apply one global palette→token map → inject the responsive scaffold); verify all screens over ONE long-lived server; LLM only on the residue | Per-screen agents re-derive the mechanical ~80% (extraction, tokenization, scaffold) N times and re-boot the dev server per check — scripting it once is the 10x, at higher reliability      |

## The Pipeline (five stages, one paid)

1. **Extract** (script): headless render under a determinism contract — `document.fonts.ready`, frozen clock, `*{animation:none;transition:none}`, fixed viewport/DPR, mouse parked. Walk _painted_ elements only (own text, visible background/border/shadow, or replaced tags). Stamp every element in-page with an index-path id, record it per node, dump the annotated `outerHTML`. Output: tree JSON (geometry spec) + HTML dump (transform input), same ids by construction.
2. **Tokenize** (script): map raw computed values through the project token map (mock brand color → kit brand color). Unmapped values flagged, never guessed. Theme-swappable tokens map as _roles_, not values (see Mistakes).
3. **Transform** (script): annotated HTML → flat JSX. `class→className`, style strings → objects, void self-closing, SVG attr camelCase, ids carried verbatim. Ugly is fine — it's scaffolding the gate holds in place.
4. **Gate** (script): extract the app the same way; pair nodes by id; diff. Tolerances: ±1px boxes/spacing, exact colors and font sizes post-mapping, both-"normal" line-heights pass. Fix the app to match the mock; **never loosen the differ**. Pixel backstop only at matched frames.
5. **Refactor** (LLM — the only paid stage): flat JSX → components, props, state, data wiring. Gate green before and after every pass.

## Common Mistakes

| Mistake                                                                                                                                                                                                                                                             | Fix                                                                                                                                                                                                               |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Generate app from prose spec, converge via defect loop (52 defects, one paid iteration each)                                                                                                                                                                        | Transform from source, refactor under the lock (pilot: 1 finding, 0 fix cycles)                                                                                                                                   |
| Paying an LLM (or vision model) to judge screenshots for conformance                                                                                                                                                                                                | Computed styles are free, deterministic ground truth; scripts check them forever at $0                                                                                                                            |
| Naive value-map on a theme-swappable token: gate flags tenant accent `#157F5B` as a defect vs kit default `#9A3412`                                                                                                                                                 | Map roles, not values: expected = the app's _resolved_ themable token, or pin the fixture tenant to kit defaults                                                                                                  |
| `el.tagName === 'SVG'` — SVG elements keep source-case (`'svg'`); every icon silently drops out of the painted set                                                                                                                                                  | Compare `tagName.toUpperCase()`; assert expected replaced-element counts in extraction acceptance                                                                                                                 |
| `pnpm --filter x dev -- --port N`: pnpm forwards the literal `--`, vite treats it as its own rest-args separator and drops the port flags — lands on the right port only by luck                                                                                    | Drop the `--`; always pass `--strictPort` so a fallback fails loudly instead of silently binding elsewhere                                                                                                        |
| Diffing absolute viewport coordinates across different shells — everything red from frame offset alone                                                                                                                                                              | Normalize boxes relative to the paired content roots                                                                                                                                                              |
| Requiring node-for-node DOM mirroring — gate fights every kit substitution and refactor                                                                                                                                                                             | Pair and diff _painted_ elements only; unpainted wrappers are free to differ                                                                                                                                      |
| The syntactic transform preserves the mock's inline styles; raw colors/durations inline in the emitted TSX bypass a filename-based `*-mock.css` lint exemption (which only covers CSS files)                                                                        | Hoist inline color/duration declarations into the exempt `-mock.css`, keyed by class or `data-mock-id`; re-run the tree-diff to prove the relocation is lossless                                                  |
| `clock.setFixedTime` pins `Date` but lets `setInterval` keep firing — live countdowns/timers diff non-deterministically between two runs                                                                                                                            | `clock.install({ time })` before navigation freezes timers too; freeze mock and app identically so the timer shows the same value in both                                                                         |
| Comparing text content — mock fixture names vs app data differ legitimately                                                                                                                                                                                         | Compare properties, never strings; classify text-bearing widths as fluid                                                                                                                                          |
| Recreating measured x/y/w/h with absolute positioning (or a scale ratio) for "pixel-perfect"                                                                                                                                                                        | Dead code at any other viewport; carry layout _intent_ (flex, gap, tokens), verify with _resolved_ geometry                                                                                                       |
| A `both`-filled CSS animation on a shell ancestor keeps its computed `transform` a matrix (never `none`) forever, silently making it the containing block for `position:fixed` descendants — full-bleed/modal screens get boxed into the ancestor, not the viewport | Portal full-bleed/fixed screens to `document.body` (`createPortal`); the tree-diff catches it as a wrong-origin bbox, not a mystery                                                                               |
| One global hex→hex value-map can't serve a mock hex that maps to two roles (e.g. brand-primary-active AND info-text share a hex) once both are painted                                                                                                              | Confirm only one role is actually painted (safe), else use a selector/pair-map escape hatch — the concrete form of "map roles, not values"                                                                        |
| Pixel-cloning a fixed desktop mock's inline styles into an app that must be responsive — fixed widths and non-collapsing grids don't reflow (188px mobile overflow, cascading from the clone)                                                                       | The mock is a desktop SPEC, not a layout source: build fluid (grid `fr` at desktop → `1fr` at the breakpoint), add a mobile no-overflow gate; screenshot at 390px — the tree-diff (one viewport) can't see reflow |
| Re-extracting the deterministic mock, re-deriving the color→token map, and re-injecting the responsive scaffold per screen/agent — the mock is static and the palette is global; the answers are identical every screen                                             | Batch them once: extract-all + transform-all + one palette→token map + scaffold. Agents act on pre-built specs, not raw renders                                                                                   |
| Re-booting the dev server (Vite + sign-in + nav) separately for the desktop diff, the mobile check, the screenshot, and every fix iteration — minutes of pure boot per screen                                                                                       | One long-lived server + browser for the whole run; iterate fixes over HMR (edit → auto-reload → re-extract, no reboot); do all three checks in one session                                                        |
| Hand-bucketing the same uniform frame offset (shell gutter/pinned padding) as "not real" on every screen — 100–200 bbox findings per screen the human re-triages                                                                                                    | Make the differ frame-aware: subtract the known content-root offset, report only real deltas. Then green means green and nobody buckets                                                                           |

## Checklist

- [ ] Extraction determinism: fonts awaited, clock frozen, motion killed, viewport+DPR fixed, mouse parked
- [ ] Painted-element filter includes SVG/replaced tags (tagName case handled)
- [ ] Ids stamped in-page before the dump; transform carries them into JSX verbatim
- [ ] Boxes diffed relative to content roots, never viewport
- [ ] Tolerance policy written down: ±1px geometry, exact colors/sizes post-mapping
- [ ] Theme-swappable tokens mapped as roles, not literal values
- [ ] Invariant/fluid classification wired for text-bearing nodes
- [ ] Differ self-test passes: reference tree vs itself = zero findings
- [ ] Pixel backstop runs only at matched frames (reference reshaped/retinted first)
- [ ] LLM refactor prompts state the contract: gate green before and after
- [ ] Responsive builds: fluid layout + a mobile no-overflow gate (tree-diff checks one viewport only); screenshot at mobile width to catch reflow/overlap
- [ ] At app scale: extraction + transform + tokenization batched once; one long-lived server for all verification; differ subtracts the known frame offset so only real deltas surface

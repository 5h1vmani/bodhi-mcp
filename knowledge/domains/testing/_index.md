# Testing Domain

> Test strategies, automation, and quality assurance.

## Quick Summary

This domain covers how to test your code effectively: what to test, how to test it, and how to automate testing.

## Files in This Domain

No playbooks currently in this domain.

## Key Principles

1. **Test pyramid** - More unit tests, fewer E2E tests
2. **Test behavior, not implementation** - Tests survive refactoring
3. **Fast feedback** - Quick tests run often
4. **Deterministic tests** - No flaky tests
5. **Test the right things** - Coverage isn't everything

## When to Consult This Domain

- Setting up testing for a new project
- Deciding what level of testing to use
- Writing meaningful tests
- Debugging flaky tests
- Improving test coverage strategically
